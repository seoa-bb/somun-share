소문설계 1기 · 2조 · 3주차 조별 과제

「글래스 코드」 소문 확산 — 매체 실행

3주차 과제 = 매스 1가지 + 온 1가지. (콘텐츠 지시서는 개인 과제로) · 조장 김서아 · 배영순 · 하지우 · 박지호

관통하는 한 줄 (강사) — 소문거리는 ‘더 나은 이야기’가 아니라 ‘선이 그어지는 다른 이야기’. 매체는 핵심 소문 문장 “잔으로 인연을 만드는 곳” 하나를, 접점마다 형태만 바꿔 전달하는 일이다.

12주차 발표 피드백 반영 (수정)

2주차 (수정 전)소문 문장 “잔 하나로 취향을 읽고 대화의 문을 여는 바” — 길고 설명적
3주차 (수정 후) ⭐소문 문장 “잔으로 인연을 만드는 곳.” (강사 추천 — 짧고 직관적)
  • 오브제 디벨롭 — 취향 카드(입장권)에 ‘매칭 카드(찢어 나눠 갖기)’ 추가. 둘만 맞춰지는 상징물 → ‘이거 뭐야?’ 대화를 유발하는 소문거리. (강사 추천)
  • 운영 디테일 — 스탠딩 바(룸 X). ★ 스탠딩 이유(강사): 앉아 있으면 다른 자리로 옮기기 어려워 말 걸기가 부담 → 스탠딩이라야 자연스럽게 다가가고 예의 있게 거절도 쉽다(자리 이동이 자유로워야 매칭이 돈다). + 드레스 코드 필수(반바지·슬리퍼·나시 입장 불가) + 잔 가격으로 격(퀄리피케이션) 유지.
  • 논란거리 = 욕망 자극 — “싱글만?·유부남녀는?” 갑론을박이 오히려 소문(강사: 시장에 전달되면 대박).

2회의에서 정한 것 (요약)

매스 = 검색(플레이스 소개글) + 공간(테이블·벽면)
온 = 취미 모임 ‘호스트’를 매체로
  • 매스 — 강사님이 우리 조에 직접 추천: 네이버 플레이스 소개글에 가치를 소비자 언어로. 매장 안에선 같은 문장을 공간 매체(테이블 안내문·벽면)로 이어받는다.
  • 온 — 핵심 타겟(30대 후반~40대 후반 싱글)이 속한 취미 기반 모임의 호스트(주최자)를 매체로 삼아 전염병식 확산.

3과제 ① 매스 — 공간 매체 : 테이블 위 코드 시스템

온으로 데려온 손님이 매장에서 직접 만나는 접점 = 매스. 매스 5가지(검색·공간·상품·사람·콘텐츠) 중 공간 매체를 골랐다. 다만 벽면 포토존처럼 ‘사진 찍는 장치’가 아니라, 손님이 실제로 말 걸고·거절하고·다시 만남을 약속하게 만드는 ‘테이블 위 코드 시스템’으로 설계했다.

왜 포토존이 아닌가 — 손님 입장에서 “나 인연 만들러 온 바에 왔다”는 사진을 대놓고 올리긴 부담스럽다. 그래서 손님이 반드시 마주하는 테이블 위에 집중했다(테이블 = 실제 행동이 일어나는 곳).

① 취향 코드 스탠드 — 말을 걸어도 되는 신호

테이블 위 취향 코드
  • Wine / Golf / Travel / Book / Business / Quiet — 손님은 본인 취향 코드를 골라 잔 옆에 세워둔다.
  • 단순 장식 X → “이 주제로 말 걸어도 된다”는 신호. 낯선 사람에게 말 거는 부담을 줄이고 취향을 명분으로 대화가 시작된다. (예: “트래블 코드시네요, 최근에 어디 다녀오셨어요?”)

② 예의 있는 거절 카드

카드 뒷면 (거절 장치)
  • “오늘은 조용히 마시고 싶어요. 대화는 다음 잔에.”
  • 말로 거절하면 민망하지만 카드를 뒤집어두면 상대도 분위기 안 해치고 물러난다. → 핵심 가치 ‘예의 있게 거절하는 곳’을 실제 공간에서 증명.

③ 찢어 나누는 매칭 카드 ★ 핵심 소문 장치

매칭 카드 (통했을 때)
  • 연락처 대신 카드를 찢어 나눠 갖기 — “잔이 통했다면, 연락처보다 먼저 다음 잔을 약속하세요. 다시 만날 날: ___ / 시간: ___ / 장소: Glass Code”
  • 찢어진 카드 = 두 사람만 맞춰지는 상징물. 누가 보면 “이게 뭐야?” → 그 질문에서 “잔으로 인연을 만드는 곳” 소문이 시작. (+ 다음 만남도 글래스 코드에서 = 재방문)
★ 매스의 핵심 — 글래스 코드의 매스는 포토존이 아니라, 손님이 반드시 마주하는 테이블 위에서 취향을 드러내고·대화를 시작하고·예의 있게 거절하고·다음 만남을 약속하게 만드는 ‘행동 설계’다. → 온으로 모으고 → 테이블 위 코드로 경험시키고 → 찢어진 카드로 다시 퍼지게 한다.
보조 매체 — 검색(플레이스 소개글, 강사 추천: “품격 있게 매칭되는 취향 바”)·사람(직원이 코드 시스템 사용법 안내)도 같은 문장으로 함께 쓴다.
팩트 vs 픽션 — ‘품격 있는 진짜 솔로만 모이는 곳’은 말이 아니라 운영으로 증명(드레스 코드·잔 가격·코드 시스템). 실제로 그렇게 운영해야 ‘내가 하는 행위’ = 팩트.

4과제 ② 온 — 취미 모임 ‘호스트’를 매체로

온 = 사냥. 모든 사람에게 뿌리지 않고 — 취미 모임의 호스트 한 명을 매체로 만들어, 그 모임 전체 → 또 다른 모임으로 기하급수로(전염병) 퍼뜨린다.

① 타겟 → 속한 집단 → 의사결정권자

타겟 페르소나
사회적으로 자리 잡은 30대 후반~40대 후반 싱글.
속한 집단
(취미 기반)
★ 주력 = ‘소모임’ 앱 ★ — 취향 기반으로 모임을 만드는 플랫폼이라 글래스 코드와 결이 가장 가깝다. 협약 + 직접 개설 두 방향 다 가능(아래 ② ③).
보조 — CEO 조찬모임(배영순 대표 실제 멤버 → 협약 시 네트워크 활용), 취미 동호회 등.
의사결정권자
(여론 주도자)
둘 중 하나 — ⓐ 모임 주최자(호스트·동아리 회장, 장소·뒤풀이를 정하는 사람) 또는 ⓑ 영향력자(유행에 민감·센스 있어 “저 사람이 추천하면 가볼 만하다”고 사람들이 인정하는 멤버). ★ 배영순 대표가 바로 이 영향력자 포지션.
왜 딱 맞나 — 글래스 코드 = 취향(취미) 기반 만남 → 취미 모임과 결이 완전히 같다. 그래서 ‘억지 끼워맞춤’이 아니라 호스트가 데려오기 가장 자연스러운 장소가 된다.

② 협약(콜라보) 관계 — 서로의 욕망을 주고받기

우리가 호스트에게 (GIVE)
  • 모임 단체 혜택(웰컴 잔·프라이빗 스탠딩 존)
  • ‘취향 매칭 나이트’ 진행 (그날 호스트 모임만의 이벤트)
  • 호스트를 ‘특별한 자리를 만든 주최자’로 세워줌(격·인기 ↑)
호스트가 우리에게 (GET)
  • 단체 방문 + 회원에게 입소문
  • 정기 모임·뒤풀이 장소화 → 재방문
  • 회원에게 새 인연·새 회원 영입 명분 제공
핵심 — 단순 ‘단체 할인’(거래)으로 가면 약하다. 호스트의 욕망(좋은 회원 영입 + 격 있는 모임)을 채워 호스트가 영웅이 되게 만들어야 온이 작동. (강사 5②: 의사결정권자의 욕망 + 나의 욕구 결합)

③ 어떻게 닿을까 (컨택) — 강사 3가지

  • 네트워크 ⭐ — 배영순 대표가 실제 CEO 조찬모임 멤버 → 그 안 취미 분과 호스트에게 바로 제안 (이미 네트워크 있음 = 팩트).
  • 앱 — ‘소모임’·넷플연가 등 취미 모임 호스트에게 DM·콜라보 제안.
  • 콜드메일 — 동호회 운영진에게 (100개 보내면 2개 답장, 그 2개가 계기).
  • ＋ 없으면 만든다 (자기 채널) ⭐ — 기존 호스트를 기다리지 않고, ‘소모임’ 앱에 우리가 직접 취미 소모임을 개설(예: 위스키·와인 취향 모임)해 우리가 호스트 = 매체가 된다. 그 모임을 바로 글래스 코드로 모은다. (글래스 코드 = 취향 기반이라 직접 여는 게 자연스러움)
  • 지속 — 분기별 최소 1회 반복 (소문은 같은 자극을 계속 줘야 한다).

＋개인 과제 (각자 매장)

자기 업에 매스 1가지 + 온 1가지 + 콘텐츠 지시서(5가지)까지. (전파할 소문 문장 / 느낄 감정 / 보여줄 증거 2+ / 금지 표현 ‘맛있다X→다르다O’ / 행동 제한)
